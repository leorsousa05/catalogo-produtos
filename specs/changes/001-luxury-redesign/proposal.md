# 001 — Redesign de luxo e expansão do catálogo

## Motivação

O catálogo atual entrega as funcionalidades obrigatórias do professor, mas ainda é uma single-page simples. O usuário quer elevar a experiência para algo visualmente impressionante, no estilo galeria particular de luxo/curadoria, com navegação multi-página, carrinho de compras, filtros e ordenação.

## Escopo

- Redesign visual completo (estilo luxo/curadoria).
- Navegação multi-página com header/footer compartilhados.
- Página de detalhes do produto própria.
- Carrinho de compras com persistência localStorage.
- Filtro por categoria e ordenação por preço.
- Páginas Sobre e Contato.
- Manutenção do sistema de favoritos existente.

## Restrições

- HTML, CSS (Bootstrap 5), JavaScript Vanilla.
- DummyJSON API via `fetch()`.
- localStorage para estado local.
- Sem frameworks JS.
- GitHub Pages será configurado manualmente pelo usuário.

## Fora de escopo

- Backend real.
- Pagamentos.
- Autenticação real.
- GitHub Pages ativação.
